The Go Programming Language

2023-11-24
未分类
4.3k 词
  • Go
  • Study Notes

1. 入门

1.1 hello, world

  1. Go 中的包类似于其他编程语言中的库，由一个或多个 .go 源文件构成，其中 main 包用于定义一个独立的可执行程序而不是库

  2. Go 原生支持 Unicode：使用 4 byte 的 int 类型（rune）存放 Unicode

  3. 在编译的时候，特定符号后面的换行符会被转换为分号

1.2 命令行参数

  1. 习惯上，在 import <package name> 之前添加注释，对包进行解释

    • 声明函数之前同理，而且可以被 go doc 和 godoc 工具定位并作为文档显示

  2. 显式声明：var s string（强调）

  3. 变量定义了之后一定要使用

  4. 短变量声明：s := ""（不在包级别的变量中使用）

  5. i++ 和 i-- 是语句而不是表达式

  6. for 后面如果只跟一部分的话，则这一部分代表 condition（for condition == while condition）

  7. 每一个运算 / 逻辑操作符（+）都对应一个赋值操作符（+=）

1.3 找出重复行

  1. map

    • map 在执行 store, get, test 时的开销为常数时间

    • map 的 key 是能够使用 == 进行比较的任意类型（所以一些复杂类型不能用作 key ?）

    • 对 map 的 key 进行迭代时，顺序是随机的

  2. bufio.Scanner 类型：能够以行 or 单词为单位划分输入

  3. .Scan() 每次读取一行输入，并删除末尾的换行符，当读取结束时返回 false

  4. .Text() 用于获取读取的文本

  5. fmt 中以 ln 结尾的函数使用 %v 来格式化参数（%v：该类型的默认格式）

  1. 一种简单的错误处理方式
  2. 避免资源泄露
  3. 函数的声明可以在调用之后

1.4 GIF 动画

  1. 导入由多段路径组成的包时，使用路径的最后一段来引用这个包
  2. 常量
    • 常量只能是布尔值、字符串、数值
    • 常量允许定义了以后不使用

1.5 获取一个 URL

1.6 并发获取多个 URL

  1. 通道允许某一 goroutine 向另外一个传递消息，main 函数在一个 goruntine 中进行

  2. 当一个 goroutine 试图在通道上进行信息的发送或接收时，其会被阻塞；只有另一个 goroutine 开始进行接受或发送时，值的传递才会发生

1.7 一个 Web 服务器

  1. 以 / 结尾的处理模式会匹配所有含该前缀的 URL
  2. 匹配的 URL 都交付于 handler 函数进行处理
  1. Go 允许一条简单的语句写在 condition 的前面（在这里可以缩小 err 的作用域）
  1. 匿名函数

1.8 其他内容

  • 控制流

    1. 从上到下第一个匹配的 case 语句被执行

    2. default 可以不是最后一个语句

    3. 与 for 和 if 类似，在 condition 前面可以加一个简单语句

  • 命名类型

  • 方法和接口：一个关联了类型的函数被称作方法

2. 程序结构

2.1 名称

  • 如果一个声明位于函数外，那么它在整个包内都可见
  • 用命名第一个字母的大小写区分是否对包外可见（大写 → 导出）
  • 包名通常用小写字母
  • 命名使用“驼峰式”风格
  • ASCII 和 HTML 要么全部大写，要么全部小写：htmlProxy or HTMLProxy

2.2 声明

  • 声明包括：变量、常量、类型、函数

2.3 变量声明

  • 接口和引用类型（slice，指针，map，通道，函数）的零值为 nil
  • 数组和结构体的零值是所有成员的零值

2.3.1 短变量声明

  • var 声明通常用于之后才进行赋值或初值不重要的变量

  • 通常只在类似于 for 循环初始化的场景中才使用对多个变量进行短变量命名的方式

  • 短变量命名也可以用于部分赋值

  • 短变量声明至少要声明一个新变量
